GhostPay
User Guide
A privacy-first token transfer protocol. Send tokens to anyone without exposing the recipient address on-chain. Simple, non-custodial, and chain-agnostic.
Non-Custodial Commitment Scheme Auto-Detect Deposits Multi-Chain Ready No Registration
📖 Overview
How It Works
GhostPay uses a cryptographic commitment scheme to completely decouple sender and recipient on-chain.
1
Sender generates a secret code
A random 32-byte secret is generated locally in your browser. A commitment hash is derived from the secret combined with the recipient address, token, and amount. Only the commitment hash is ever stored on-chain — never the secret itself.
2
Sender deposits tokens into the contract
Tokens are locked in the smart contract against the commitment hash. The deposit transaction records only the commitment hash — no recipient address, no readable amount correlation.
3
Secret code is shared off-chain
The sender shares the secret code with the recipient through any private off-chain channel — Telegram DM, Signal, email, or any other secure method. This is the only piece of information the recipient needs.
⚠️ Never share the secret code publicly or on-chain. Keep it private.
4
Recipient claims the transfer
The recipient connects their wallet and pastes the secret code. GhostPay automatically scans on-chain events to find the matching deposit. Once confirmed, tokens land directly in their wallet.
🔒 Security
Privacy Model
Understanding what is and is not visible on-chain.
● Not visible on-chain
The secret code, recipient address, and readable amount do not appear in the withdrawal calldata or event logs. No direct on-chain link is created between sender and recipient.
● Visible on-chain
The sender address (in the deposit transaction), the commitment hash, and the deposit deadline are publicly visible. The token type and locked amount are stored in contract state.
● Important limitation
An analyst who has the secret code can re-derive the commitment hash and correlate sender to recipient. Keeping the secret private is essential to maintaining privacy.
● Double-spend protection
Each secret code can only be used once. A nullifier hash is permanently stored on-chain after a successful claim, preventing any replay or double-claim attacks.
💡 For maximum privacy, use a fresh wallet address as the recipient that has no prior on-chain history linking it to your identity.
📤 User Guide
Sending a Transfer
Step-by-step guide to sending a private token transfer.
1
Connect your wallet
Click Connect Wallet in the top-right corner. GhostPay supports MetaMask and any compatible Web3 wallet. The app will prompt you to switch to the correct network if needed.
2
Fill in the transfer details
Open the Send tab. Select the token you want to send, enter the amount, and paste the recipient's wallet address.
3
Set a refund delay
Choose how long before you can cancel and reclaim the funds if the recipient never claims. This protects you in case the recipient loses the secret code or never acts on it.
4
Generate secret code and deposit
Click Generate Secret Code. For ERC-20 tokens, you must approve the contract first — then confirm the deposit transaction in your wallet. Save the Secret Code, Commitment Hash, and Nullifier shown on screen.
Save all three values before closing this screen. They cannot be recovered if lost.
5
Share the secret code privately
Send the Secret Code to the recipient through a private channel. They only need this code and their matching wallet to claim the funds.
📥 User Guide
Claiming a Transfer
How to claim tokens that have been sent to you.
1
Connect the recipient wallet
Connect the wallet address that was used as the recipient by the sender. The token claim will go directly to this connected wallet — make sure it is the correct one before proceeding.
2
Paste the secret code
Open the Receive tab and paste the secret code shared by the sender. GhostPay automatically scans on-chain events across all supported tokens to find the matching deposit.
3
Review and confirm
If a matching deposit is found, the token and amount are displayed automatically. Verify the details and click Claim Tokens to confirm the transaction in your wallet.
4
Tokens arrive in your wallet
After the transaction confirms, tokens are transferred directly to your connected wallet. A link to the transaction on the block explorer is shown for reference.
ℹ️ A small amount of the native gas token is required to pay the claim transaction fee. The amount depends on current network conditions.
⚠️ If no deposit is found, verify: you are using the correct recipient wallet, the secret code is complete and unmodified, and the deposit has not already been claimed or cancelled.
↩️ User Guide
Cancel & Get a Refund
Reclaim your deposit if the recipient has not claimed within the refund delay period.
ℹ️ Cancellation is only available after the refund delay you set at deposit has fully elapsed. Only the original sender wallet can cancel.
1
Locate your Commitment Hash
The Commitment Hash was shown when you generated the secret code. It is also visible in your deposit transaction on the block explorer. Keep this saved along with your secret code.
2
Open the Cancel tab
Navigate to Cancel & Refund and paste the Commitment Hash. The app will show the current deposit status and whether the refund deadline has passed.
3
Submit the cancellation
Once the refund deadline has passed and the deposit is still unclaimed, click Cancel & Get Refund. Tokens are returned to your original sender wallet.

❓ Support
Frequently Asked Questions
Common questions about using GhostPay.
What happens if I lose the secret code?+
The secret code is the only way to claim a deposit. If it is lost, the only recovery path is to wait for the refund delay to expire and cancel the deposit to reclaim your tokens. There is no other recovery mechanism — keep the secret code saved securely.
Can someone else claim the transfer if they have the secret code?+
No. The commitment is derived from secret + recipient address + token + amount. The claim function uses msg.sender as the recipient — meaning only the wallet matching the original recipient address can successfully claim, even if another party has the secret code.
Does the recipient need the native gas token to claim?+
Yes. A small amount of the network's native token is required to pay the gas fee for the claim transaction. The exact amount depends on current network congestion and gas prices at the time of claiming.
What is the Commitment Hash?+
A cryptographic fingerprint derived from: secret code + recipient address + token address + amount. It is stored on-chain at deposit time and used to verify a claim — without ever revealing the underlying secret. Keep it saved; you will need it if you ever want to cancel the deposit.
Can I send to my own wallet address?+
Yes. You can set your own address as the recipient. This is a common use case for transferring between your own wallets without creating a direct on-chain link between the two addresses.
Why does the scan take a moment when I paste the secret?+
GhostPay scans on-chain deposit events for all supported tokens to find your matching deposit automatically. Scan time depends on the number of deposits on-chain and the RPC response speed. Make sure your wallet is connected before scanning.
Can the contract owner access deposited funds?+
No. GhostPay is fully non-custodial. The contract owner can only manage the supported token list. Only the recipient (with the correct secret code and wallet) or the original depositor (after the refund delay expires) can ever move funds.
Will GhostPay support more networks?+
Yes. GhostPay is built to be chain-agnostic and is designed to expand to additional EVM-compatible networks. The current deployment is for testing purposes. More networks and tokens will be added over time.
